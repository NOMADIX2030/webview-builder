# 앱 링크(App Links) 딥링크 확장 기능 가이드

> App Links 서버 설정 완료 후 활용 가능한 확장 기능 목록.
> 웹뷰 앱의 기능을 네이티브 수준으로 확장하는 방법을 정리한 문서.
>
> **전제 조건**: `https://도메인/.well-known/assetlinks.json` 서버 배포 완료

---

## 1. 딥링크 마케팅 (즉시 활용)

### 개요
이메일, 카카오톡, SMS, SNS 등에 URL을 공유할 때 앱이 설치된 사용자는 앱에서, 미설치 사용자는 웹에서 해당 페이지가 열립니다.

### 활용 예시
```
특정 이벤트 페이지 공유
https://도메인/event/summer-sale

특정 상품 페이지 공유
https://도메인/product/123

채팅방 초대 링크
https://도메인/chat/room/456
```

### 앱에서 처리
현재 구현된 `onNewIntent`에서 딥링크 URL을 WebView에 그대로 로드합니다.
추가 개발 없이 바로 동작합니다.

### 권한
불필요

---

## 2. QR 코드 → 앱 직접 연결

### 개요
QR 코드에 `https://도메인/경로`를 삽입하면, 스마트폰 카메라로 스캔 시 앱이 열리고 해당 화면으로 바로 이동합니다.

### 활용 예시
```
명함 QR → 담당자 프로필 페이지
현수막 QR → 이벤트 페이지
테이블 QR → 주문 페이지
포스터 QR → 제품 소개 페이지
```

### 동작 흐름
```
카메라로 QR 스캔
→ 앱 설치 O: 앱이 열리며 해당 페이지 표시
→ 앱 설치 X: 브라우저로 웹 페이지 표시
```

### 권한
불필요 (QR 스캔은 기기 카메라 앱이 처리, 우리 앱 권한 불필요)

## 3. 소셜 로그인 자동 복귀 확장

### 개요
카카오 외 다른 소셜 로그인(네이버, 구글 등) 인증 완료 후 자동으로 앱으로 복귀합니다.

### 지원 가능 서비스
| 서비스 | callback URL 패턴 | 비고 |
|---|---|---|
| 카카오 | `https://도메인/auth/kakao/callback` | 현재 적용됨 |
| 네이버 | `https://도메인/auth/naver/callback` | assetlinks.json 배포 시 동작 |
| 구글 | `https://도메인/auth/google/callback` | assetlinks.json 배포 시 동작 |
| 기타 OAuth | `https://도메인/auth/*/callback` | 동일 패턴 |

### 동작 원리
이미 AndroidManifest에 `android:host="{도메인}"` 전체가 등록되어 있어 해당 도메인의 모든 URL이 앱으로 라우팅됩니다. 별도 개발 없이 assetlinks.json 배포만으로 동작합니다.

### 권한
불필요

## 4. 앱 전용 기능 트리거 (딥링크 → 네이티브 기능)

### 개요
웹 또는 외부에서 특정 URL을 열면 앱의 네이티브 기능이 실행됩니다.

### 구현 방법

**현재 `onNewIntent`에 URL 패턴 분기 추가:**

```java
@Override
protected void onNewIntent(Intent intent) {
    super.onNewIntent(intent);
    setIntent(intent);
    Uri data = intent.getData();
    if (data != null) {
        String path = data.getPath();
        if ("/open-camera".equals(path)) {
            // 카메라 기능 실행
        } else if ("/scan-qr".equals(path)) {
            // QR 스캔 실행
        } else {
            // 일반 딥링크: WebView에 URL 로드
            webView.loadUrl(data.toString());
        }
    }
}
```

### 기능별 권한 및 구현 난이도

| 기능 | URL 예시 | 필요 권한 | 난이도 |
|---|---|---|---|
| 특정 페이지 이동 | `/page/event` | 없음 | ⭐ |
| 카메라 열기 | `/open-camera` | `CAMERA` | ⭐⭐ |
| QR 스캔 | `/scan-qr` | `CAMERA` | ⭐⭐ |
| 전화 연결 | `/call/01012345678` | `CALL_PHONE` | ⭐⭐ |
| 지도/위치 | `/map?lat=37.5&lng=127.0` | `ACCESS_FINE_LOCATION` | ⭐⭐⭐ |
| 파일 다운로드 | `/download/file.pdf` | (Android 10+ 불필요) | ⭐⭐ |

### 권한 요청 시점
앱 설치 시가 아닌 **해당 기능을 처음 사용할 때** 팝업으로 요청됩니다.

### 빌더에서 권한 추가 방법
Step 2 → 추가 권한 선택 체크박스에서 필요한 권한 선택 후 빌드.

## 5. 푸시 알림 → 정확한 화면 이동

### 개요
FCM 푸시 알림 클릭 시 앱의 특정 페이지로 정확하게 이동합니다. App Links와 결합하면 앱이 완전히 종료된 상태(cold start)에서도 동작합니다.

### 서버 FCM 데이터 구조
```json
{
  "action_url": "https://도메인/chat/room/123",
  "title": "새 메시지",
  "body": "안녕하세요",
  "unread_count": "5"
}
```

### 이동 가능한 페이지 예시
```
채팅방: https://도메인/chat/room/123
주문 내역: https://도메인/orders/456
알림 상세: https://도메인/notifications/789
이벤트 페이지: https://도메인/event/summer
```

### 권한
불필요 (FCM 권한은 별도 — 알림 권한 `POST_NOTIFICATIONS`)

## 6. 웹 ↔ 앱 전환 배너

### 개요
모바일 웹에서 "앱으로 보기" 버튼을 제공하여 앱 설치 사용자는 앱으로, 미설치 사용자는 스토어로 유도합니다.

### 웹 개발자 구현 예시

```javascript
// 앱 설치 여부 감지 후 딥링크 시도
function openInApp() {
    const appUrl = 'https://도메인' + window.location.pathname;
    const storeUrl = 'https://play.google.com/store/apps/details?id=com.패키지명';

    // 딥링크 시도
    window.location.href = appUrl;

    // 일정 시간 후 앱이 안 열리면 스토어로 이동
    setTimeout(() => {
        window.location.href = storeUrl;
    }, 2000);
}
```

### 권한
불필요

## 7. 구현 우선순위 및 로드맵

| 우선순위 | 기능 | 추가 개발 필요 | 비고 |
|---|---|---|---|
| 즉시 활용 | 딥링크 마케팅 | ❌ 없음 | assetlinks.json 배포만 하면 됨 |
| 즉시 활용 | QR 코드 연결 | ❌ 없음 | QR에 URL만 삽입 |
| 즉시 활용 | 소셜 로그인 복귀 | ❌ 없음 | assetlinks.json 배포만 하면 됨 |
| 단기 | 앱 전용 기능 트리거 | ✅ 앱 코드 수정 | onNewIntent URL 분기 추가 |
| 단기 | 웹↔앱 전환 배너 | ✅ 웹 코드 추가 | 웹 개발자 작업 |
| 중기 | 카메라/QR 스캔 | ✅ 네이티브 기능 추가 | Capacitor 플러그인 활용 |

## 8. 주의사항

### assetlinks.json 관리
- 앱 재빌드 시 새 Keystore가 생성되면 SHA256이 변경됩니다.
- SHA256 변경 시 assetlinks.json도 함께 업데이트해야 합니다.
- 빌드 완료 페이지에서 SHA256을 확인하고 서버에 재배포하세요.

### Keystore 보관
- 동일 SHA256을 유지하려면 같은 Keystore로 앱을 서명해야 합니다.
- Play Store 배포 후에는 Keystore를 절대 분실하면 안 됩니다.
- Google Play App Signing 사용 시 Google이 Keystore를 관리하므로 안전합니다.

### Android 버전 호환성
| Android 버전 | App Links 동작 |
|---|---|
| Android 6.0 (API 23) 이상 | App Links 정식 지원 |
| Android 6.0 미만 | 브라우저 또는 앱 선택 팝업 표시 |
