## 1. 서비스 개요

### 1.1 목적

웹사이트 소유자가 **도메인 주소와 옵션 정보만 입력**하면, 해당 웹을 **웹뷰 앱** 또는 **하이브리드 앱**으로 패키징하여 **APK/IPA 빌드 파일**을 생성해 주는 서비스.

- **빌드 파일 생성**: 서비스가 담당
- **앱스토어 출시**: 사용자가 직접 수행

---

### 1.2 서비스 구조

| 구분 | 제공 방식 | 대상 | 주요 기능 |
|------|-----------|------|-----------|
| **일반 웹뷰** | 무상 | 단순 URL 로드 앱 | 웹 URL 로드, 기본 앱 셸 |
| **하이브리드 앱** | 유료 | 네이티브 기능 필요 앱 | 푸시(필수) + 위치/카메라/파일 등 |
| **커스텀 개발** | 별도 견적 | 범용 옵션으로 불가능한 요구 | 맞춤형 하이브리드 앱 개발 |

## 2. 상세 서비스 내용

### 2.1 무상 — 일반 웹뷰

| 항목 | 내용 |
|------|------|
| **대상** | 웹 URL만 앱으로 감싸고 싶은 사용자 |
| **기능** | URL 로드, 앱 아이콘, 스플래시, 기본 설정 |
| **입력** | 1단계: 도메인, 로고 / 2단계: 자동 생성값 수정 / 3단계: 시뮬레이션 확인 |
| **출력** | Android APK, iOS IPA |
| **가격** | 무료 |

---

### 2.2 유료 — 하이브리드 앱

| 항목 | 내용 |
|------|------|
| **대상** | 푸시, 위치, 카메라 등 네이티브 기능이 필요한 사용자 |
| **필수 기능** | 푸시 알림 |
| **선택 기능** | 위치 기반, 카메라, 갤러리, 파일 핸들링 등 |
| **입력** | 1단계: 도메인, 로고, 푸시 설정, 네이티브 옵션 / 2단계: 자동 생성값 수정 / 3단계: 시뮬레이션 확인 |
| **출력** | Android APK, iOS IPA |
| **가격** | 유료 |

---

## 3. 기술 스택

### 3.1 개발 스택

| 계층 | 기술 | 역할 |
|------|------|------|
| **프론트엔드** | Next.js + shadcn/ui | 3단계 폼 UI, 모바일 우선 디자인 |
| **백엔드** | Laravel | API, 빌드 큐, 파일 저장 |
| **빌드 엔진** | Capacitor | 웹뷰/하이브리드 앱 생성 |
| **DB** | MariaDB | 빌드 이력, 메타데이터 |

**디자인 표준**: shadcn/ui 전용, 모바일 우선, 네이티브 앱 수준. 상세는 DEV_SPEC.md 2.3 참조.

### 3.2 Capacitor (앱 빌드)

| 구분 | 내용 |
|------|------|
| **선택 이유** | 웹 기반, 템플릿 주입 용이, 웹뷰/하이브리드 모두 지원 |
| **공식 사이트** | https://capacitorjs.com |
| **문서** | https://capacitorjs.com/docs |

### 3.3 지원 기능 (생성 앱)

| 기능 | 대응 | 비고 |
|------|------|------|
| WebSocket | ✅ | WebView에서 그대로 동작 |
| 실시간 채팅 | ✅ | 기존 웹 로직 그대로 사용 |
| 푸시 알림 | ✅ | Capacitor 플러그인 |
| 위치 | ✅ | Geolocation 플러그인 |
| 카메라/갤러리 | ✅ | Camera 플러그인 |

## 4. 사용자 입력 흐름 (3단계)

### 4.1 입력 원칙

| 원칙 | 내용 |
|------|------|
| **최소 입력** | 도메인 주소, 로고 등 필수 이미지만 수집 |
| **자동 생성** | 나머지 필수 정보는 서비스가 추천값으로 자동 생성 |
| **수정 가능** | 자동 생성된 모든 항목은 사용자가 수정 가능 |
| **확인 후 진행** | 3단계에서 최종 확인 후에만 빌드 진행 |

---

### 4.2 1단계 — 최소 정보 수집

| 항목 | 내용 |
|------|------|
| **목적** | 필수 정보만 최소한으로 수집 |
| **입력** | 도메인 주소, 로고(앱 아이콘), (선택) 스플래시 이미지 |
| **하이브리드 시** | 푸시 설정, 네이티브 기능 옵션 등 추가 |
| **다음** | 2단계로 이동 |

---

### 4.3 2단계 — 자동 생성 + 수정

| 항목 | 내용 |
|------|------|
| **목적** | 1단계 정보를 기반으로 나머지 필수 정보 자동 생성 |
| **표시** | 앱 이름, 패키지 ID, 개인정보처리방침 URL, 문의 URL, 버전 등 추천값 |
| **동작** | 모든 항목 수정 가능 |
| **다음** | 3단계로 이동 |

---

### 4.4 3단계 — 시뮬레이션 + 최종 확인

| 항목 | 내용 |
|------|------|
| **목적** | 실제 앱 모습 확인 후 최종 확정 |
| **표시** | 모바일 화면 시뮬레이션 (로고, 스플래시, 웹 로드 화면 등) |
| **동작** | 최종 확인 후 빌드 진행 |
| **다음** | 빌드 실행 → APK/IPA 생성 → 다운로드 제공 |

---

### 4.5 전체 흐름도

```
[1단계] 최소 정보 수집
   도메인, 로고, (선택) 스플래시, 하이브리드 옵션
        ↓
[2단계] 자동 생성 + 수정
   1단계 기반 추천값 표시 → 사용자 수정 가능
        ↓
[3단계] 시뮬레이션 + 최종 확인
   모바일 화면 미리보기 → 확인 → 빌드 진행
        ↓
[빌드] APK/IPA 생성 → 다운로드 → 사용자가 앱스토어 출시
```

## 5. 입력 정보 정의

### 5.0 원칙

| 원칙 | 내용 |
|------|------|
| **최소 입력** | 1단계에서 도메인, 로고 등만 수집 |
| **자동 생성** | 2단계에서 나머지 필수 항목 추천값 생성 |
| **수정 가능** | 사용자가 모든 자동 생성값 수정 가능 |
| **출시 대비** | 앱스토어 출시 시 문제 없도록 모든 필수 항목 수집 |

### 5.0.1 자동 생성 매핑 (1단계 → 2단계)

| 1단계 입력 | 2단계 자동 생성 추천값 |
|------------|------------------------|
| 도메인 (예: https://myplatform.com) | 앱 이름: Myplatform |
| | 패키지 ID: com.myplatform.app |
| | 개인정보처리방침 URL: https://myplatform.com/privacy |
| | 문의/지원 URL: https://myplatform.com/contact |
| (공통) | 버전: 1.0.0, versionCode: 1 |
| (공통) | Android Keystore: 서비스에서 자동 생성 후 다운로드 제공 |

---

### 5.1 일반 웹뷰 (공통 필수)

| 단계 | 필수 | 항목 | 설명 | 출시 시 필요성 |
|------|------|------|------|----------------|
| 1 | ✅ | 웹 URL | 로드할 도메인 | - |
| 1 | ✅ | 앱 아이콘 | 1024x1024 (iOS), 512x512 (Android) | 스토어 필수 |
| 1 | △ | 스플래시 스크린 | 선택 | 앱 첫 실행 UX |
| 2 | ✅ | 앱 이름 | 앱스토어 표시명 (도메인 기반 자동 생성) | 스토어 리스팅 |
| 2 | ✅ | 패키지 ID | 예: com.company.appname (도메인 기반 자동 생성, 변경 불가) | 앱 고유 식별 |
| 2 | ✅ | **개인정보처리방침 URL** | 활성 URL (도메인 기반 자동 생성) | 양 스토어 필수 |
| 2 | ✅ | **문의/지원 URL 또는 이메일** | 고객 문의용 (도메인 기반 자동 생성) | 양 스토어 필수 |
| 2 | ✅ | **버전 정보** | versionName (1.0.0), versionCode (1) 자동 생성 | 빌드·업데이트 시 필수 |

---

### 5.2 앱스토어 출시용 (공통)

| 필수 | 항목 | 설명 | 비고 |
|------|------|------|------|
| ✅ | **Android 서명** | Keystore 파일 또는 서비스에서 생성 후 전달 | Play Store 업로드용. 업데이트 시 동일 keystore 필요 |
| △ | **iOS 서명** | Apple Developer 계정 연동 또는 프로젝트 제공 | IPA 서명. 사용자 로컬 빌드 또는 CI 연동 |
| △ | 화면 방향 | portrait / landscape / both | 앱 설정 |
| △ | 딥링크(URL Scheme) | 푸시 탭, 공유 시 이동할 URL 스킴 | 예: myapp:// |

---

### 5.4 권한·개인정보 (하이브리드 앱 필수)
