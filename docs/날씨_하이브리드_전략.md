# 날씨 하이브리드 전략

> **목적**: 실시간·정확한 서울 날씨를 여러 소스를 조합해 안정적으로 제공  
> **작성일**: 2026-03-11  
> **관련**: WeatherExtractService, weather-section.blade.php

---

## 1. 전략 요약

| 순위 | 소스 | API 키 | 정확도 | 비고 |
|------|------|--------|--------|------|
| **1** | Open-Meteo | 불필요 | 높음 | KMA(기상청) 데이터, 10,000회/일 |
| **2** | 기상청 공공데이터 | 필요 | 최고 | 자동승인, .env에 키 설정 시 |
| **3** | Groq AI | 필요 | 낮음 | 폴백, 어제 대비 등 보강용 |

**폴백 체인**: 1 실패 → 2 시도 → 2 실패 → 3 폴백

---

## 2. 1순위: Open-Meteo

### 특징
- **API 키 불필요**: 즉시 사용 가능
- **서울 지원**: lat=37.5665, lon=126.9780
- **KMA 연동**: Open-Meteo가 기상청(KMA) 모델 사용 (Seoul: LDPS 1.5km, GDPS 12km)
- **제한**: 10,000회/일, 5,000회/시간
- **응답**: JSON, current + hourly

### API 엔드포인트
```
GET https://api.open-meteo.com/v1/forecast
  ?latitude=37.5665
  &longitude=126.9780
  &timezone=Asia/Seoul
  &current=temperature_2m,relative_humidity_2m,apparent_temperature,precipitation,weathercode,cloud_cover,wind_speed_10m,wind_direction_10m,visibility,pressure_msl
  &hourly=temperature_2m,relative_humidity_2m,apparent_temperature,precipitation_probability,precipitation,weathercode,cloud_cover,wind_speed_10m,uv_index
  &daily=temperature_2m_max,temperature_2m_min,uv_index_max,precipitation_sum,precipitation_probability_max,weathercode
```

### AI 브리핑 (Open-Meteo 1순위일 때만)
- Open-Meteo 상세 데이터를 Groq AI에 전달하여 **날씨 브리핑** 생성
- 포함 내용: 옷차림 추천, 기상 변동 예측(오전/오후/저녁), 주의사항(UV·바람·강수 등)
- `GROQ_API_KEY` 설정 시 활성화, 실패 시 빈 문자열(표시 안 함)

### WMO weathercode → 한글 매핑
| code | 한글 | 이모지 |
|------|------|--------|
| 0 | 맑음 | ☀️ |
| 1 | 대체로 맑음 | ⛅ |
| 2 | 약간 흐림 | ⛅ |
| 3 | 흐림 | ☁️ |
| 45,48 | 안개 | 🌫️ |
| 51-67 | 비/소나기 | 🌧️ |
| 71-77 | 눈 | ❄️ |
| 80-82 | 소나기 | 🌦️ |
| 85,86 | 눈 소나기 | ❄️ |
| 95-99 | 뇌우 | ⛈️ |

### 없음
- `yesterday_diff` (어제 대비): → AI 보강 또는 생략

## 3. 2순위: 기상청 공공데이터 API

### 조건
- `.env`에 `DATA_GO_KR_SERVICE_KEY` 설정 시 활성화
- 공공데이터포털(data.go.kr) 회원가입 → 활용신청(자동승인)

### API
- `getVillageFcst`: 단기예보 (3시간 간격)
- `getUltraSrtFcst`: 초단기예보 (1시간)

---

## 4. 3순위: Groq AI (폴백)

- Open-Meteo·기상청 모두 실패 시 기존 방식 유지
- `yesterday_diff` 등 부가 정보는 AI로 보강 가능 (선택)

---

## 5. AI 활용 포인트 (선택)

| 항목 | Open-Meteo | AI 보강 |
|------|------------|---------|
| 현재 기온 | ✅ | - |
| 시간별 기온 | ✅ | - |
| 날씨 상태 | ✅ (weathercode) | - |
| 어제 대비 | ❌ | ✅ (AI 추론 또는 전날 동시각 비교) |

**어제 대비**: Open-Meteo Historical API로 전날 동시각 조회 가능 (선택 구현)

## 6. 구현 흐름

```
get()
  → Cache hit? return
  → try fetchOpenMeteo()
       success → normalize → cache → return
  → try fetchDataGoKr() (if key set)
       success → normalize → cache → return
  → fetchFromGroq() (fallback)
       → normalize → cache → return
  → emptyResult()
```

## 8. 위치 기반 (2026-03-11)

- **초기 로드**: 서울 좌표 (37.5665, 126.9780)
- **Geolocation**: 브라우저 `navigator.geolocation.getCurrentPosition` → `/api/landing/weather?lat=&lon=` → 날씨 섹션 갱신
- **지역명**: Open-Meteo Geocoding API (reverse) → 좌표 → 한글 지역명
- **API**: `GET /api/landing/weather?lat=37.5&lon=127` (Accept: text/html → HTML partial 반환)
