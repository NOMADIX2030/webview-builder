# Laravel + Next.js 기술 스택 아키텍처 문서

> **작성일**: 2026년 2월 26일  
> **대상**: 웹앱 개발 (블로그, 실시간 채팅, 회원 관리, 하이브리드 앱 확장)

---

## 1. 개요

본 문서는 **Laravel 백엔드**와 **Next.js 프론트엔드**를 결합한 풀스택 아키텍처의 구조, 역할, 관계를 정의합니다. **웹 우선 런칭** 후 **Capacitor**로 구글/애플 앱 확장이 가능한 구성을 추천합니다.

### 1.1 추천 스택 요약

| 구분 | 기술 | 핵심 역할 |
|------|------|-----------|
| **백엔드** | Laravel (PHP) | API, 인증, DB, 웹소켓 서버 |
| **프론트엔드** | Next.js (React) | 페이지 렌더링, 블로그, 채팅 UI, SEO |
| **DB** | MariaDB 10.x | 관계형 데이터 저장 |
| **캐시** | Redis | 세션, 캐시, 큐 |
| **하이브리드 앱** | Capacitor | 웹 → iOS/Android 앱스토어 배포 |

## 2. 전체 아키텍처 및 관계도

### 2.1 핵심 아키텍처 다이어그램

```
┌─────────────────────────────────────────────────────────────────────────┐
│                      Next.js (프론트엔드)                                 │
├─────────────────────────────────────────────────────────────────────────┤

┌───────────────────────────▼─────────────────────────────────────────────┐

│  • API 라우트          • 인증 (Sanctum)    • 권한 관리                    │
```

## 3. Laravel 백엔드 역할

### 3.1 API 라우트

| 역할 | 설명 | 예시 |
|------|------|------|
| **REST 엔드포인트** | CRUD API 제공 | `GET /api/posts`, `POST /api/chat/messages` |
| **라우트 그룹** | 인증/권한별 미들웨어 적용 | `auth:sanctum`, `role:admin` |

```
/api/posts          - 블로그 글 목록, 상세, 생성, 수정, 삭제
/api/categories     - 카테고리 목록
/api/users          - 회원 정보 (인증 필요)
/api/chat/rooms     - 채팅방 목록
/api/chat/messages  - 메시지 목록, 전송
```

### 3.2 인증 (Authentication)

| 역할 | 설명 |
|------|------|
| **로그인** | 이메일/비밀번호, 소셜 로그인 (Google, Naver 등) |
| **토큰 발급** | Laravel Sanctum으로 API 토큰 발급 |
| **토큰 검증** | `Authorization: Bearer {token}` 헤더 검증 |

### 3.3 권한 관리 (Authorization)

| 역할 | 설명 |
|------|------|
| **역할(Role)** | Admin, Editor, Member 등 |
| **권한(Permission)** | 글 작성, 글 수정, 회원 관리 등 |
| **미들웨어** | `auth:sanctum`, `can:edit-post` 등 |

| 권한 레벨 | 접근 범위 |
|-----------|-----------|
| **Guest** | 블로그 읽기, 문의 폼 |
| **Member** | 로그인, 프로필, 채팅 참여 |
| **Editor** | 블로그 글 작성·수정 |
| **Admin** | 회원 관리, 설정, 전체 데이터 |

### 3.4 데이터베이스 (MariaDB 10.x)

| 역할 | 설명 |
|------|------|
| **ORM** | Eloquent로 모델·관계 정의 |
| **마이그레이션** | 스키마 버전 관리 |
| **접근 제어** | Policy, Scope로 쿼리 제한 |
| **Laravel 연결** | `mysql` 드라이버로 MariaDB 호환 |

| 용도 | 테이블 예시 |
|------|-------------|
| **블로그** | posts, categories, tags, comments |
| **회원** | users, roles, permissions, role_user |
| **채팅** | chat_rooms, messages, chat_participants |
| **일반 페이지** | pages, menus, settings |

### 3.5 웹소켓 (실시간 채팅)

| 구성요소 | 역할 |
|----------|------|
| **Laravel Reverb** | WebSocket 서버 (Laravel 11+ 기본 제공) |
| **Broadcasting** | 채널·이벤트 정의, 권한 검사 |
| **Pusher 프로토콜** | Reverb가 사용 (Laravel Echo 호환) |

```
[Next.js + Laravel Echo]  ←→  WebSocket  ←→  [Laravel Reverb]
        ↑                                        ↑
   채널 구독, 메시지 수신                    메시지 브로드캐스트
```

### 3.6 기타

| 역할 | 설명 |
|------|------|
| **파일 저장** | 이미지, 첨부파일 업로드 |
| **이메일** | 비밀번호 재설정, 알림 메일 |
| **큐** | 비동기 작업 (메일 발송, 리포트 생성) |
| **캐시** | Redis로 API 응답 캐싱 |

## 4. Next.js 프론트엔드 역할

### 4.1 페이지 렌더링

| 방식 | 용도 |
|------|------|
| **SSR** | SEO 중요 페이지, 동적 콘텐츠 |
| **SSG** | 정적 페이지, 블로그 글 |
| **CSR** | 대시보드, 채팅 등 실시간 UI |

### 4.2 블로그

| 역할 | 설명 |
|------|------|
| **목록** | 카테고리별, 페이지네이션 |
| **상세** | 글 본문, 목차, 관련글 |
| **SEO** | meta, JSON-LD (Article, BreadcrumbList) |

### 4.3 일반 페이지

| 역할 | 설명 |
|------|------|
| **랜딩** | 홈, 서비스 소개 |
| **회사소개** | 팀, 연혁, 연락처 |
| **문의** | 폼 제출 |

### 4.4 회원 UI

| 역할 | 설명 |
|------|------|
| **로그인/회원가입** | 폼, 유효성 검사 |
| **프로필** | 정보 수정, 비밀번호 변경 |
| **로그아웃** | 토큰 삭제, 리다이렉트 |

### 4.5 실시간 채팅

| 역할 | 설명 |
|------|------|
| **Laravel Echo** | WebSocket 연결, 채널 구독 |
| **메시지 수신** | 실시간 메시지 표시 |
| **메시지 전송** | API 호출 또는 브로드캐스트 |

### 4.6 라우팅

| 경로 | 용도 |
|------|------|
| `/` | 홈 |
| `/blog` | 블로그 목록 |
| `/blog/[slug]` | 블로그 상세 |
| `/login` | 로그인 |
| `/register` | 회원가입 |
| `/chat` | 채팅 (인증 필요) |
| `/admin/*` | 관리자 (Admin 권한) |

### 4.7 Capacitor (하이브리드 앱)

| 역할 | 설명 |
|------|------|
| **웹뷰 래핑** | Next.js 빌드 결과를 네이티브 WebView에 로드 |
| **앱스토어 배포** | iOS App Store, Google Play Store 출시 |
| **네이티브 플러그인** | 카메라, 푸시 알림, 저장소 등 확장 |

| 배포 단계 | 설명 |
|-----------|------|
| **1단계: 웹** | Next.js 웹 배포 (SSR/SSG) |
| **2단계: 앱** | `output: 'export'` → Capacitor `webDir` → iOS/Android 빌드 |

## 5. 인증·권한 흐름

```
[Next.js]  →  POST /api/login  →  [Laravel]  →  Sanctum 토큰 발급
                ↓
[Next.js]  →  API 요청 시 Authorization: Bearer {token}
                ↓
[Laravel]  →  auth:sanctum 미들웨어로 권한 검사
```

---

## 6. 웹소켓 채팅 흐름

```
1. 사용자 A가 메시지 전송
   → Next.js → POST /api/chat/messages → Laravel

2. Laravel 저장 후 브로드캐스트
   → Reverb → WebSocket → 구독 중인 클라이언트들

3. 사용자 B의 Next.js (Echo)
   → 실시간으로 메시지 수신 → UI 업데이트
```

---

## 7. 데이터 흐름 (블로그 글 조회 예시)

```
1. 사용자 요청 → Next.js (GET /blog/my-post)

2. Next.js getServerSideProps / getStaticProps
   → fetch('https://api.example.com/api/posts/my-post')

3. Laravel API
   → Post::with('category')->where('slug', 'my-post')->first()
   → JSON 응답

4. Next.js
   → HTML 렌더링 → 클라이언트 전달 (SEO 유리)
```

## 8. 배포 전략 (웹 우선 → 앱 확장)

| 단계 | 대상 | 방법 |
|------|------|------|
| **1단계** | 웹 | Next.js를 Vercel/서버에 배포 (SSR/SSG) |
| **2단계** | iOS/Android | Next.js `output: 'export'` → Capacitor webDir → 앱 빌드 |

| 환경 | Next.js 설정 | API |
|------|-------------|-----|
| **웹** | SSR/SSG | Laravel API (동일) |
| **앱** | Static Export | Laravel API (동일) |

## 12. AI 코딩 (Cursor) 관점

| 기술 | AI 코딩 적합도 | 이유 |
|------|----------------|------|
| **Laravel** | ⭐⭐⭐⭐⭐ | 관례가 명확, 문서·예제 풍부 |

| **Capacitor** | ⭐⭐⭐⭐ | 문서·예제 충분 |
| **TypeScript** | ⭐⭐⭐⭐⭐ | 타입 정보로 정확한 코드 생성 |

| 권장 | 설명 |
|------|------|
| **TypeScript 사용** | Next.js에 TypeScript 적용 |
| **`.cursorrules`** | 프로젝트 규칙·스택 명시 |
| **일관된 네이밍** | `getUserById`, `createPost` 등 규칙 유지 |
