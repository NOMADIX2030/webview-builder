> **상태**: 구현 완료 (로고·설정 UI 포함)  
> **작성일**: 2026-03-09

## 1. 문서 개요

### 1.1 이 문서의 역할

| 대상 | 용도 |
|------|------|
| **Cursor AI** | 랜딩 페이지 개발 시 참조할 핵심 명세. 이 문서의 구조, 규칙, 모듈화 원칙을 **정확히** 따를 것 |
| **개발자** | 섹션 추가·수정 시 구현 가이드 |

### 1.2 랜딩 페이지 목적

| 항목 | 내용 |
|------|------|
| **역할** | 도메인 루트(`/`)에서 여러 비즈니스 기능으로 진입하는 허브 |

| **관리** | 섹션·기능 목록을 DB 또는 설정으로 동적 관리 |

## 2. 아키텍처 원칙

### 2.1 섹션 모듈화 (필수)

> **AI 개발 시**: 랜딩 페이지는 **섹션 단위로 완전히 분리**된 구조여야 한다.

| 원칙 | 내용 |
|------|------|
| **1섹션 1파일** | 각 섹션은 독립된 Blade 컴포넌트 또는 partial 파일로 존재 |
| **의존성 최소화** | 섹션 간 직접 참조 금지. 컨트롤러/서비스에서 데이터만 주입 |
| **추가 시 기존 수정 없음** | 새 섹션 추가 시 기존 섹션 파일·코드 수정 금지 |
| **순서·노출 제어** | 섹션 순서, 노출 여부는 데이터(DB/설정)로 제어 |

### 2.3 섹션 추가 절차 (AI 체크리스트)

새 섹션 추가 시 **반드시** 다음 순서만 따를 것:

| 순서 | 작업 | 수정 대상 | 기존 파일 수정 |
|------|------|-----------|----------------|
| 1 | 섹션 partial 생성 | `views/landing/sections/{section-name}.blade.php` | 없음 |
| 2 | 설정에 등록 | `config/landing.php` 또는 DB `landing_sections` | 설정만 추가 |
| 3 | index에서 include | `views/landing/index.blade.php` | `@include` 한 줄 추가 (또는 루프로 자동) |
| 4 | (선택) 서비스 확장 | `LandingSectionService.php` | 새 섹션 데이터 반환 로직만 추가 |

**금지**: 기존 섹션 파일 내용 수정, 다른 섹션에서 새 섹션 직접 참조

### 5.1 라우팅

```php
Route::prefix('build')->name('build.')->group(function () {
    // ... 기존 유지
});
```

### 5.2 LandingController

**파일**: `app/Http/Controllers/Web/LandingController.php`

```php
<?php

namespace App\Http\Controllers\Web;

use App\Http\Controllers\Controller;
use App\Services\LandingSectionService;
use Illuminate\View\View;

class LandingController extends Controller
{
    public function __construct(
        private LandingSectionService $sectionService
    ) {}

    public function index(): View
    {
        return view('landing.index', [
            'sections' => $this->sectionService->getVisibleSections(),
            'features' => $this->sectionService->getVisibleFeatures(),
        ]);
    }
}
```

### 5.3 LandingSectionService

**파일**: `app/Services/LandingSectionService.php`

```php
<?php

namespace App\Services;

use Illuminate\Support\Collection;

class LandingSectionService
{
    public function getVisibleSections(): Collection
    {
        $config = config('landing.sections', []);
        return collect($config)
            ->where('visible', true)
            ->sortBy('order')
            ->values();
    }

    public function getVisibleFeatures(): Collection
    {
        $config = config('landing.features', []);
        return collect($config)
            ->where('visible', true)
            ->sortBy('order')
            ->values();
    }
}
```

### 5.4 랜딩 레이아웃

**파일**: `resources/views/layouts/landing.blade.php`

- `app.blade.php`와 유사하되, 랜딩 전용 스타일 적용
- `@yield('content')` 영역에 섹션들이 배치됨
- max-width는 데스크톱에서 더 넓게 (예: max-w-2xl 또는 max-w-4xl)

### 5.5 랜딩 index (섹션 조합)

**파일**: `resources/views/landing/index.blade.php`

```blade
@extends('layouts.landing')

@section('content')

        @include("landing.sections.{$section['id']}", [
```

## 7. 새 섹션 추가 예시

### 7.1 예: "최근 소식" 섹션 추가

**1단계**: partial 생성

**파일**: `resources/views/landing/sections/recent-news.blade.php`

```blade
<section class="rounded-xl border border-gray-200 bg-white p-6 dark:border-gray-700 dark:bg-gray-800">
    <h2 class="text-lg font-semibold text-gray-900 dark:text-white">최근 소식</h2>
    <p class="mt-2 text-sm text-gray-500 dark:text-gray-400">...</p>
</section>
```

**2단계**: `config/landing.php`에 추가

```php
'sections' => [
    // ... 기존
    [
        'id' => 'recent-news',
        'order' => 3,
        'visible' => true,
    ],
],
```

**3단계**: `LandingController`에서 데이터 주입 (필요 시)

```php
return view('landing.index', [
    'sections' => $this->sectionService->getVisibleSections(),
    'features' => $this->sectionService->getVisibleFeatures(),
    'recentNews' => [], // 새 섹션용 데이터
]);
```

**4단계**: `index.blade.php`에서 `$section['id']`가 `recent-news`이면 `sections/recent-news.blade.php`가 자동 include됨 (루프 구조일 경우).  
또는 `@include("landing.sections.{$section['id']}", ['recentNews' => $recentNews ?? []])` 형태로 변수 전달.

## 10. 뉴스 섹션 상세 (news-grid)

### 10.1 뉴스 소스 및 API

| 소스 | RSS URL | 언어 | 상세 페이지 |
|------|---------|------|-------------|
| 연합뉴스 | yna.co.kr/rss/news.xml | ko | 내부 스크래핑 (`/news/detail?u=`) |
| TechCrunch | techcrunch.com/category/artificial-intelligence/feed | en | 내부 스크래핑 |
| VentureBeat | venturebeat.com/category/ai/feed | en | 외부 링크 |
| MIT Review | technologyreview.com/feed | en | 외부 링크 |

### 10.2 API 엔드포인트

| 메서드 | 경로 | 설명 |
|--------|------|------|
| GET | `/api/landing/news` | 페이징 뉴스 (`page`, `category`, `q` 쿼리) |
| GET | `/api/landing/news/counts` | 탭별 뉴스 개수 |
| GET | `/api/landing/settings` | 로고·기능 카드 설정 |

### 10.3 서비스 구조

| 서비스 | 역할 |
|--------|------|
| `NewsRssService` | RSS 파싱, 캐시(10분), 발행처별 필터, 영어→한국어 번역(제목·설명) |
| `NewsScraperService` | 연합뉴스·TechCrunch 기사 본문 스크래핑, 본문 번역, 캐시(30분) |
| `TranslateService` | Stichoza/google-translate-php, 24시간 캐시, HTML 본문 단락별 번역 |

### 10.4 설정 UI

- `/settings` — 로고 이미지·텍스트 업로드, 기능 카드 CRUD
- `LandingSettingsService`, `LandingFeature` 모델, `landing_settings`·`landing_features` 테이블
