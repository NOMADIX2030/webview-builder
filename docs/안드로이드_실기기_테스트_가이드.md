# 안드로이드 실기기 테스트 가이드

> ADB를 통한 실기기 연결, 실시간 로그 분석, App Links 검증 방법을 정리한 문서.

---

## 1. 사전 준비

### 1.1 기기에서 개발자 옵션 활성화

1. **설정 → 휴대폰 정보 → 소프트웨어 정보** 이동
2. **빌드 번호**를 **7번 연속 탭**
3. "개발자가 되었습니다" 메시지 확인
4. 설정 뒤로 나오면 **개발자 옵션** 메뉴 생성됨

### 1.2 USB 디버깅 활성화

1. **설정 → 개발자 옵션** 이동
2. **USB 디버깅** 토글 ON
3. USB 케이블로 Mac에 연결
4. 기기 화면에 팝업 → **"이 컴퓨터에서 항상 허용"** 선택 후 확인

### 1.3 연결 확인

```bash
adb devices
```

정상 연결 시 출력:
```
List of devices attached
XXXXXXXXXXXXXXXX    device
```

> `unauthorized` 상태라면 기기 화면에서 디버깅 허용 팝업을 확인하세요.

## 2. APK 설치 및 App Links 검증

### 2.1 APK 직접 설치 시 주의사항

APK를 직접 설치(sideload)하면 **App Links 자동 검증이 동작하지 않습니다.**
재설치할 때마다 아래 검증 명령을 수동으로 실행해야 합니다.

> Google Play Store를 통해 배포하면 설치 시 자동 검증됩니다.

### 2.2 App Links 수동 검증 (재설치 후 매번 실행)

```bash
# 1. 검증 강제 실행
adb shell pm verify-app-links --re-verify com.패키지명

# 예시
adb shell pm verify-app-links --re-verify com.optiflow.app
```

```bash
# 2. 검증 결과 확인
adb shell pm get-app-links com.패키지명

# 예시
adb shell pm get-app-links com.optiflow.app
```

**정상 출력 예시:**
```
  com.optiflow.app:
    ID: xxxxxxxx-xxxx-xxxx-xxxx-xxxxxxxxxxxx
    Signatures: [BE:90:45:55:...]
    Domain verification state:
      optiflow.kr: verified   ← 이 상태여야 App Links 동작
```

> `verified`가 아닌 경우: `assetlinks.json` 서버 배포 여부 및 SHA256 일치 여부 확인

## 3. 실시간 로그 분석 (logcat)

### 3.1 로그 초기화

테스트 전 이전 로그를 지워 깨끗한 상태에서 시작합니다.

```bash
adb logcat -c
```

### 3.2 실시간 로그 모니터링

**특정 키워드 필터링 (권장):**

```bash
# 앱·카카오·Intent 관련 핵심 로그만 표시
adb logcat | grep -E "optiflow|kakao|VIEW|NewIntent|deeplink|AppLink|verify"
```

**노이즈 제거 포함 버전:**

```bash
adb logcat | grep -E "optiflow|kakao|VIEW|NewIntent|deeplink|AppLink|verify" \
  | grep -v "setRequestedFrameRate\|frameRate\|SurfaceComposer\|setCornerRadius"
```

**앱 프로세스 로그만 표시:**

```bash
# 패키지명으로 PID 확인 후
adb shell pidof com.optiflow.app

# 해당 PID 로그만
adb logcat --pid=$(adb shell pidof com.optiflow.app)
```

### 3.3 로그 파일로 저장

```bash
adb logcat > ~/Desktop/test_log.txt
# 테스트 완료 후 Ctrl+C
```

### 3.4 실시간 모니터링 종료

터미널에서 **`Ctrl + C`** 입력

## 4. 로그 사후 분석

### 4.1 저장된 로그에서 특정 키워드 검색

```bash
# 로그 초기화 없이 최근 로그만 덤프
adb logcat -d | grep -E "optiflow|kakao|Intent" \
  | grep -v "setRequestedFrameRate\|SurfaceComposer" | tail -100
```

### 4.2 주요 로그 패턴 해석

| 로그 내용 | 의미 |
|---|---|
| `Focus entered window: com.optiflow.app/...MainActivity` | 우리 앱이 포그라운드로 전환됨 |
| `freeze com.optiflow.app(...) result: 2` | 앱이 백그라운드(freeze) 상태 |
| `freeze com.optiflow.app(...) result: 12` | 앱이 완전 freeze (메모리 절약) |
| `onNewIntent` 호출 | App Links 또는 딥링크 Intent 수신 |
| `com.kakao.talk/.../ChatRoomHolderActivity` | 카카오톡 2차 인증 화면 실행 |
| `optiflow.kr: verified` | App Links 검증 완료 |

## 5. App Links 동작 확인 테스트

### 5.1 테스트 순서

```
1. APK 재설치
2. adb shell pm verify-app-links --re-verify com.패키지명
3. adb shell pm get-app-links com.패키지명 → verified 확인
4. adb logcat -c (로그 초기화)
5. adb logcat | grep -E "optiflow|kakao|Intent|NewIntent" 실행
6. 앱에서 카카오 로그인 시도
7. 인증 완료 후 Ctrl+C
8. 로그에서 onNewIntent 호출 여부 확인
```

### 5.2 성공 판단 기준

- `onNewIntent` 로그 존재 + 앱이 자동으로 포그라운드 복귀 → **App Links 동작 성공**
- 수동 전환 후 로그인 완료 상태 → **로그인 자체는 성공, 복귀만 없음**

## 6. 카카오 2차 인증 관련 주의사항

### 6.1 동작 특성

카카오 2차 인증(카카오톡 알림 방식)은 카카오톡 앱 내부(`ChatRoomHolderActivity`)에서 인증을 완결합니다.

```
[2차 인증 없는 계정]
카카오톡에서 승인 → callback URL을 외부로 열기 시도
→ App Links가 동작 → 우리 앱 자동 복귀 ✅

[2차 인증 있는 계정]
카카오톡 내부에서 인증 완결 → callback URL 외부 미전달
→ App Links 개입 불가 → Android 10+ 정책상 자동 복귀 불가
→ 수동 앱 전환 필요 (정상 동작 범위)
```

### 6.2 결론

2차 인증 계정의 수동 복귀는 **OS 레벨 한계**로 앱 단에서 해결 불가합니다.
수동 복귀 후 로그인이 완료된 상태라면 정상 동작입니다.

## 7. assetlinks.json 확인

### 7.1 서버 배포 확인

```bash
curl https://도메인/.well-known/assetlinks.json
```

### 7.2 올바른 형식

```json
[{
  "relation": ["delegate_permission/common.handle_all_urls"],
  "target": {
    "namespace": "android_app",
    "package_name": "com.패키지명",
    "sha256_cert_fingerprints": ["XX:XX:XX:..."]
  }
}]
```

### 7.3 SHA256 fingerprint 확인

빌드 완료 페이지(`/build/{id}`)의 **App Links 설정** 섹션에서 복사 버튼으로 획득합니다.

## 8. 자주 발생하는 문제 및 해결

| 증상 | 원인 | 해결 |
|---|---|---|
| `adb: no devices/emulators found` | USB 디버깅 미허용 또는 케이블 불량 | 기기에서 디버깅 허용 팝업 확인, 케이블 교체 |
| `unauthorized` | 기기에서 허용 미클릭 | 기기 화면 팝업에서 "항상 허용" 선택 |
| `optiflow.kr: 1024` | assetlinks.json 미배포 또는 검증 실패 | 서버 배포 확인, SHA256 일치 여부 확인 |
| App Links 동작 안 함 | APK 재설치 후 재검증 미실행 | `verify-app-links --re-verify` 재실행 |
